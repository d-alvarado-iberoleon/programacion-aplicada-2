Introducción (paradigmas de programación)

Existen muchos paradigmas (estilos genéricos) de programación:

  • Programación Declarativa
  • Programación Imperativa
  • Programación Estructurada
  • Programación Lógica
  • Programación Funcional
  • Programación Orientada a Objetos

Programación Declarativa

  • Describe qué debe hacerse, sin especificar cómo lograrlo.
  • Se enfoca en el resultado, no en el proceso.
  • Los lenguajes SQL y HTML son ejemplos de este paradigma.
SELECT nombre, edad FROM estudiantes WHERE edad > 18;

Programación Imperativa

  • Describe una serie de instrucciones que se ejecutan en un orden específico para cambiar el estado del programa.
  • Se enfoca en cómo realizar tareas.
  • Usa variables mutables y estructuras de control como bucles y condicionales.
  • Es el enfoque más tradicional y ampliamente utilizado.
numeros = [1, 2, 3, 4]
suma = 0
for numero in numeros:
    suma += numero ** 2
print(suma)  
30

Programación Estructurada

  • Es un paradigma que organiza el código en bloques lógicos para hacerlo más fácil de entender, mantener y depurar. Se basa en tres estructuras principales:
    • Secuencia: Instrucciones ejecutadas en orden.
    • Condición: Decisiones condicionales como if-else.
    • Iteración: Repetición mediante bucles como for o while.
calificacion = float(input("Ingresa una calificación (0-100): "))

if 0 <= calificacion <= 100:
    print("Calificación válida")
else:
    print("La calificación debe estar entre 0 y 100.")

print("Fin del programa.")

Programación Lógica

  • Se basa en la lógica formal y trabaja definiendo hechos, reglas y consultas.
  • En lugar de decirle al programa cómo realizar una tarea, se especifica qué condiciones deben cumplirse.
% Hechos
padre(juan, maria).
padre(juan, pedro).

% Regla
hermanos(X, Y) :- padre(P, X), padre(P, Y), X \= Y.

% Consulta
?- hermanos(maria, pedro).  % Resultado: true

Programación Funcional

  • Utiliza funciones, donde cada función produce el mismo resultado si se le dan los mismos argumentos.
  • Las funciones pueden asignarse a variables o pasarse como argumentos.
# Función
def suma_cuadrados(lista):
    return sum(x**2 for x in lista)

numeros = [1, 2, 3, 4]
print(suma_cuadrados(numeros))  # Resultado: 30
30

Programación Orientada a Objetos

  • Organiza el código en objetos, que combinan datos (atributos) y comportamientos (métodos). Este paradigma se enfoca en modelar entidades del mundo real.
class Rectangulo:
    def __init__(self, base, altura):
        self.base = base
        self.altura = altura

    def calcular_area(self):
        return self.base * self.altura

rectangulo = Rectangulo(5, 10)
print(rectangulo.calcular_area())
50

Nota importante

(en la sesión anterior estuvimos usando los paradigmas imperativo, estructurado y funcional)

  • Los programas se ejecutan de forma secuencial (una instrucción tras otra).
  • Lo único que puede desviar el flujo del programa son las estructuras condicionales (como if-elif-else), las estructuras de repetición (como for, while) o las funciones que encapsulan bloques de código reutilizable.
  • Estos paradigmas se enfocan en el control preciso del flujo del programa, lo que significa que el programador debe especificar exactamente cómo deben ejecutarse las instrucciones.

(ahora abordaremos el paradigma orientado a objetos)

Programación Orientada a Objetos

  • La Programación Orientada a Objetos es un paradigma que consiste en modelar sistemas a través de la creación de objetos que contienen atributos y métodos.
  • Tiene como objetivos el facilitar el desarrollo de software robusto y adaptable a medida que el proyecto crece y mejorar el mantenimiento del software mediante la estructuración en clases, facilitando la comprensión y modificaciones.

Pilares

Para cumplir los objetivos anteriores, se utilizan los siguientes pilares, que explicaremos durante esta sesión.

  1. Abstracción: Simplifica la representación de un objeto, centrándose en aspectos esenciales.
  2. Encapsulamiento: Oculta detalles internos, exponiendo solo interfaces necesarias.
  3. Polimorfismo: Capacidad de objetos de diferentes clases de comportarse de manera uniforme (interfaz común).
  4. Herencia: Crear nuevas clases basadas en clases existentes.

Clases y objetos

Hemos visto que Python ofrece distintos tipos de datos para crear variables (int, float, String, etc), pero también ofrece la posibilidad de crear nuestros propios tipos de datos para modelar conceptos más complejos.

¿Qué es una clase?

Note

Es una plantilla para definir un tipo de dato compuesto por atributos (variables) y métodos (funciones/comportamientos).

¿Qué es un Objeto?

Note

  • Es un valor o variable cuyo tipo de dato es compuesto, es decir es elemento que tiene atributos y métodos.
  • Más técnicamente, un objeto es una instancia de una clase en la memoria del programa.

Algunos ejemplos y analogías

Podemos pensar en una clase como un cortador de galletas y en un objeto como una galleta en particular.

¿Cómo crear una clase y un objeto en Python?

Sintaxis

Para crear una clase, se utiliza la palabra clave class, seguida del nombre de la clase y dos puntos.

class MiClase:
    pass  # pass no realiza ninguna acción, se usa para indicar que aquí pondremos más código en el futuro

La creación de un objeto se realiza mediante la asignación de una instancia de la clase a una variable. Es decir, escribimos el nombre que tendrá el objeto a crear, el símbolo = y el nombre de la clase (tipo de dato del objeto) seguido de paréntesis.

mi_objeto = MiClase()

por ejemplo…

¿Qué es un constructor?

  • Un constructor es un método especial en una clase que se llama automáticamente cuando se crea un objeto de esa clase.

    (un método es una función dentro de la clase)

  • En Python, el constructor se llama __init__ (doble guión bajo al inicio y al final) y se utiliza para inicializar los atributos del objeto.

  • El constructor puede tener cualquier número de parámetros, incluso ninguno.

  • Por ejemplo, en nuestra clase Estudiante podemos agregar algunos atributos (variables) que se crearán al crear un nuevo objeto.

# Define una nueva clase (plantilla/tipo de dato/plano) llamada "Estudiante"
class Estudiante:
    def __init__(self, nombre, no_cuenta, asignaturas = []):  # Constructor de la clase
        self.nombre = nombre
        self.no_cuenta = no_cuenta
        self.asignaturas = asignaturas

# Crea un objeto de la clase "Estudiante"
estudiante1 = Estudiante("Ana Laura Cabrera", "192344-I")

# Imprime su tipo de dato
print(type(estudiante1))  # Imprime <class '__main__.Estudiante'>

La palabra self

Important

  • La palabra self se refiere al objeto que está siendo creado o manipulado dentro de los métodos de la clase.
  • Cuando se llama a un método de una clase, la instancia (objeto) en la que se invoca ese método se pasa automáticamente como el primer argumento, y se nombra convencionalmente como self.
  • Aunque self es solo una convención y podría llamarse de otra manera, su uso es altamente recomendado para mantener un código claro y fácil de entender.

¿Qué son los atributos?

  • Son variables que pertenecen a un objeto específico de una clase.
  • Representan características o propiedades del objeto.
  • En Python, los atributos pueden ser de dos tipos principales:
    • atributos de instancia
    • atributos de clase.
Tipo de atributo Definición Acceso desde objeto ¿Se comparte entre objetos?
De instancia Dentro de __init__ con self. obj.atributo ❌ No
De clase Dentro de la clase, fuera de métodos Clase.atributo o obj.atributo ✅ Sí

Atributos de instancia

  • Son específicos de cada instancia de la clase.
  • Cada objeto tiene su propia copia del atributo de instancia.
  • Se definen dentro de los métodos de la clase: generalmente en el método __init__, utilizando el prefijo self.
# Define una nueva clase (plantilla/tipo de dato/plano) llamada "Estudiante"

        self.no_cuenta = no_cuenta  # Atributo de instancia
        self.asignaturas = asignaturas

¿Qué son los métodos?

  • Son funciones que están definidas dentro de esa clase y actúan sobre los atributos que pertenecen a la misma.
  • Representan acciones o comportamientos del objeto.
  • Hay tres diferentes tipos de métodos que puedes tener en una clase de Python:
    • Métodos de instancia
    • Métodos de clase
    • Métodos estáticos

Métodos de instancia

  • Se definen con el primer parámetro como self, que hace referencia a la instancia de la clase en sí.
  • Acceden y manipulan los atributos de la instancia.
# Define una nueva clase (plantilla/tipo de dato/plano) llamada "Estudiante"
class Estudiante:

    def __init__(self, nombre, no_cuenta, asignaturas = []):  # Constructor de la clase, "self" representa la instancia actual
        self.nombre = nombre  # Atributo de instancia
        self.no_cuenta = no_cuenta  # Atributo de instancia
        self.asignaturas = asignaturas

    def agregar_asignatura(self, asignatura):
        self.asignaturas.append(asignatura)
        print(f"{self.nombre} ha agregado la asignatura {asignatura}.")

    def informacion(self):
        info = f'Nombre: {self.nombre}\n'
        info += f'Número de cuenta: {self.no_cuenta}\n'
        info += 'Asignaturas:\n'
        for asignatura in self.asignaturas:
            info += asignatura + '\n'
        return info

estudiante1 = Estudiante("Ana Laura Cabrera", "192344-I")
estudiante2 = Estudiante("Juan Enrique Pérez", "182621-M")

# Métodos de instancia
estudiante1.agregar_asignatura("Física")  # Salida: Juan ha agregado la asignatura Física.

print(estudiante1.informacion())

Encapsulamiento

  • Como hemos visto, en Python podemos acceder a los atributos de clase con la palabra reservada self desde dentro de la clase, y con el nombre del objeto/clase, punto, y nombre del atributo desde el exterior (en el programa principal, módulo o en otra clase).

¿Qué es el encapsulamiento?

  • Es uno de los pilares de la programación orientada a objetos (OOP).
  • Se refiere a la ocultación de los detalles internos de una clase y la restricción del acceso directo a ciertos componentes.

¿Por qué hacer encapsulamiento?

  • Protege un objeto del acceso no deseado mediante código fuera de la clase. EJ: Una aplicación bancaria prohíbe a un cliente cambiar el saldo de una Cuenta.
  • Permite cambiar/actualizar la implementación de la clase si es necesario, sin que eso afecte el funcionamiento de la misma. EJ: Podríamos cambiar la lógica detrás de un método, pero su nombre y parámetros no cambian, así que se pueden seguir utilizando de la misma manera.
  • Permite restringir el estado de los objetos (atributos invariantes).

Atributos y métodos públicos

  • Los atributos y métodos públicos son accesibles desde fuera de la clase.
  • Los atributos y métodos cuyos nombres no comienzan con dos guiones bajos se consideran públicos.

Atributos y métodos privados

  • Un atributo o método se considera privado cuando su nombre comienza con dos guiones bajos (__).
  • Estos atributos y métodos no son accesibles directamente desde fuera de la clase.

POR EJEMPLO…

class CuentaBancaria:
    def __init__(self, titular, saldo_inicial=0):
        # Atributos privados
        self.__titular = titular
        self.__saldo = saldo_inicial

    # Método privado para validar si la cantidad es positiva
    def __es_cantidad_valida(self, cantidad):
        if cantidad <= 0:
            print("Error: La cantidad debe ser mayor que cero.")
            return False
        return True

    # Método privado para verificar si hay fondos suficientes
    def __hay_fondos_suficientes(self, cantidad):
        if cantidad > self.__saldo:
            print("Error: Fondos insuficientes.")
            return False
        return True

    # Método público para depositar dinero
    def depositar(self, cantidad):
        if self.__es_cantidad_valida(cantidad):
            self.__saldo += cantidad
            print(f"Depósito exitoso. Nuevo saldo: ${self.__saldo:.2f}")

    # Método público para retirar dinero
    def retirar(self, cantidad):
        if self.__es_cantidad_valida(cantidad) and self.__hay_fondos_suficientes(cantidad):
            self.__saldo -= cantidad
            print(f"Retiro exitoso. Nuevo saldo: ${self.__saldo:.2f}")

cuenta.depositar(500)  # Depósito exitoso
cuenta.retirar(200)    # Retiro exitoso

cuenta.retirar(-100)   # Error: La cantidad debe ser mayor a cero

Métodos Getter y Setter

Para acceder o modificar atributos privados, se deben implementar métodos llamados getters y setters.

  • Un getter es un método que obtiene el valor de un atributo privado.
  • Un setter es un método que establece el valor de un atributo privado.

Agreguemos getters y setters para la clase CuentaBancaria:

Opción 1 (método tradicional)

    def depositar(self, cantidad):
        if self.__es_cantidad_valida(cantidad):

Name Mangling

  • Python realiza un cambio de nombre (name mangling) de los atributos privados al nivel de la clase, agregando un prefijo _NombreDeLaClase antes del nombre del atributo.
  • Esto dificulta el acceso a estos atributos desde fuera de la clase, pero no lo impide completamente.
class Ejemplo:
    def __init__(self):
        self.__atributo_privado = 0

objeto = Ejemplo()
# Acceso directo al atributo privado utilizando el nombre mangling
print(objeto._Ejemplo__atributo_privado)

Polimorfismo

  • Etimológicamente, “polimorfismo” significa “muchas formas”.
  • El polimorfismo es un principio de la POO que permite que distintos objetos respondan a la misma operación (función o método) de maneras diferentes.
  • En POO, el polimorfismo se manifiesta de varias maneras:
    1. Con métodos
    2. Con sobrecarga de operadores
    3. Con herencia

Imágenes de stayfreemagazine.org

¿Por qué usar polimorfismo?

  • Permite usar una misma interfaz para objetos de diferentes clases.
  • Facilita la extensibilidad del código.
  • Reduce el uso de condicionales y verificaciones de tipo.
  • Fomenta el uso de estructuras abstractas y genéricas.
Tipo Descripción Ejemplo en Python
Polimorfismo de método Métodos con el mismo nombre se comportan distinto en distintas clases area() en Rectángulo, Círculo, etc.
Polimorfismo por herencia Clases derivadas redefinen (sobrescriben) métodos de la clase base __str__() redefinido en subclases
Sobrecarga de operadores Permite redefinir operadores como +, *, ==, etc. usando métodos mágicos __add__(), __eq__(), __mul__()

1. Polimorfismo con métodos

  • Una clase puede implementar un método con el mismo nombre que otra clase, pero con un comportamiento diferente.
  • Esto permite que objetos de distintas clases respondan a la misma llamada de método de manera diferente.
import math

class Cuadrado:
    def __init__(self, lado):
        self.lado = lado
    def area(self):
        return self.lado**2

class Rectangulo:
    def __init__(self, base, altura):
        self.base = base
        self.altura = altura
    
    def area(self):
        return self.base*self.altura
        
class Circulo:
    def __init__(self, radio):
        self.radio = radio
    def area(self):
        return math.pi * self.radio**2

fig1 = Cuadrado(5)
fig2 = Rectangulo(3, 4)
fig3 = Circulo(2)

lista = [fig1, fig3, fig2]
for fig in lista:
    print(fig.area())

2. Polimorfismo con sobrecarga de operadores

  • Es una forma de polimorfismo que permite a los objetos responder a operadores de manera diferente según el contexto.
  • La acción de sobrecargar operadores se refiere a la implementación de métodos especiales en las clases que creamos, conocidos como métodos mágicos o métodos dunder.

Los métodos mágicos o métodos dunder son métodos especiales que tienen nombres que comienzan y terminan con doble guion bajo. Estos métodos son utilizados para realizar operaciones específicas en objetos y son invocados automáticamente por Python en situaciones particulares.

Método Descripción Ejemplo de Uso
__str__(self) Representación en cadena del objeto str(obj), print(obj)
__len__(self) Devuelve la longitud del objeto len(obj)
__add__(self, otro) Sobrecarga del operador de suma (+) obj1 + obj2
__sub__(self, otro) Sobrecarga del operador de resta (-) obj1 - obj2
__mul__(self, otro) Sobrecarga del operador de multiplicación (*) obj1 * obj2
__truediv__(self, otro) Sobrecarga del operador de división (/) obj1 / obj2
__eq__(self, otro) Sobrecarga del operador de igualdad (==) obj1 == obj2
__ne__(self, otro) Sobrecarga del operador de desigualdad (!=) obj1 != obj2 
class Cuadrado:
    def __init__(self, lado):
        self.lado = lado
    def area(self):
        return self.lado**2
    def __eq__(self, otro):
        if isinstance(otro, Cuadrado):
            return self.lado == otro.lado
    def __str__(self):
        return f'Cuadrado(lado: {self.lado})'

fig4 = Cuadrado(5)
fig5 = Cuadrado(6)
print(fig4 == fig5)
print(fig4)
False
Cuadrado(lado: 5)

Herencia

  • La herencia es un mecanismo de la POO que permite a una clase adquirir atributos y métodos de otra clase.
  • Se utiliza para modelar relaciones jerárquicas entre clases.

Clase base

  • Las clases existentes se llaman clases base. Son las que proporcionan los atributos y métodos.
  • También reciben otros nombres como superclase, clase padre o antepasado.

Clase derivada

  • La clase que hereda de otra se llama clase derivada.
  • También se llama con otros nombres como subclase, clase secundaria y descendiente.
  • La clase derivada hereda los atributos/métodos de la clase base y normalmente contiene atributos/métodos adicionales.

La función super

  • La función super() es una función incorporada que se utiliza para acceder a métodos y atributos de una clase base.
  • Se puede usar para llamar a métodos y atributos de la clase base sin tener que especificar explícitamente el nombre de la clase base.
class Persona:
    def __init__(self, nombre):
        self.nombre = nombre

    def __str__(self):
        return f'Persona (nombre: {self.nombre})'

class Estudiante(Persona):
    def __init__(self, nombre, carrera):
        super().__init__(nombre)  # llama al constructor de Persona
        self.carrera = carrera
    def __str__(self):
        return f'Estudiante (nombre: {self.nombre}; carrera: {self.carrera})'

estudiante1 = Estudiante("Raúl Rojas", "Ing. en Robótica")
print(estudiante1)
Estudiante (nombre: Raúl Rojas; carrera: Ing. en Robótica)

Tipos de herencia

Tipo Descripción Ejemplo
Simple Una clase hereda de una sola clase base class B(A): ...
Múltiple Una clase hereda de dos o más clases base class C(A, B): ...
Multinivel Una clase hereda de otra que también hereda class C(B); class B(A)
Jerárquica Varias clases heredan de una misma clase base class B(A); class C(A)

Nota:

  • La herencia múltiple puede dar lugar a conflictos de nombres, en caso de que los padres tengan métodos con el mismo nombre.
  • Python sigue el orden de izquierda a derecha para buscar atributos o métodos en las clases base.
  • Si dos clases base tienen un método con el mismo nombre, el método de la clase más a la izquierda se utilizará.
# Clase base 1
class ClaseBase1:
    def metodo_comun(self):
        return "Método común de ClaseBase1"
    
# Clase base 2
class ClaseBase2:
    def metodo_comun(self):
        return "Método común de ClaseBase2"

# Clase derivada que hereda de ambas clases bases
class ClaseDerivada(ClaseBase1, ClaseBase2):
    def metodo_derivado(self):
        return "Método específico de ClaseDerivada"

# Uso de la clase derivada
instancia_derivada = ClaseDerivada()

# Intentamos llamar al método común, pero hay ambigüedad
print(instancia_derivada.metodo_comun())

¿Por qué usar herencia?

  • Para evitar repetir código (reutilización).
  • Para crear estructuras jerárquicas más claras.
  • Para permitir especialización progresiva (la subclase redefine o amplía).

Abstracción

  • La abstracción es el principio de la POO que permite ocultar los detalles internos de implementación de un objeto y exponer solo lo necesario para su uso.
  • En otras palabras, permite modelar lo esencial de una entidad y ocultar lo irrelevante o complejo para el usuario del objeto.
    • Un auto: puedes encenderlo y conducirlo sin saber cómo funciona el motor internamente.
    • Una cafetera: basta con oprimir un botón; no necesitas conocer el sistema de calentamiento ni presión.
    • En ambos casos, el usuario interactúa con una interfaz simple y el funcionamiento interno está abstraído.

Clases abstractas

Python permite la abstracción principalmente mediante clases abstractas. Una clase abstracta:

  • Es una clase que no puede ser instanciada directamente y generalmente sirve como una plantilla para clases derivadas.
  • Se define con la ayuda del módulo abc (Abstract Base Classes).
  • Puede contener métodos abstractos, que son métodos sin implementación, dejando que las clases derivadas proporcionen sus implementaciones.

En el siguiente ejemplo, Figura es una clase abstracta que define un método abstracto area. Las clases derivadas (Rectangulo, Cuadrado y Circulo) deben proporcionar implementaciones concretas para este método.

  • La creación de una instancia de la clase abstracta directamente generaría un error.
from abc import ABC, abstractmethod

class Figura(ABC):
    @abstractmethod
    def area(self):
        pass

class Rectangulo(Figura):
    def __init__(self, base, altura):
        self.base = base
        self.altura = altura

    def area(self):
        return self.base * self.altura

Intentar instanciar la clase abstracta genera un error:

fig = Figura()  # generará un error

Pero sí podemos crear objetos de las clases derivadas:

rectangulo1 = Rectangulo(5, 4)
print(rectangulo1.area())
20

¿Cuándo usar abstracción?

  • Usa abstracción cuando tengas múltiples clases que compartirán una estructura común.
  • Evita instanciar clases abstractas directamente.
  • Nombra tus métodos abstractos como parte de una interfaz clara (por ejemplo: area(), dibujar(), guardar()).
Beneficio Descripción
Claridad de uso El usuario del objeto solo ve lo esencial.
Evita duplicación de código Las clases base definen interfaces comunes para varias subclases.
Facilita el mantenimiento El código interno se puede cambiar sin afectar al usuario.
