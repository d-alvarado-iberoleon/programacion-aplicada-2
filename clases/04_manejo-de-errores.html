Motivación: manejo de errores

  • Algunas veces los programas o aplicaciones que ejecutamos en la computadora enfrentan situaciones inesperadas o no válidas.
  • ¿Qué sucede en el programa?

Motivación: manejo de errores

  • El programa o aplicación puede generar una excepción, que se clasifica en varios tipos:
  1. De valor (Value Error): Ocurre cuando se proporciona una entrada de datos inesperada o inválida.
resultado = int("abc")
  1. De índice (IndexError): Se produce al intentar acceder a elementos más allá de los límites de una lista.
lista = [1, 2, 3]
elemento = lista[3]
  1. De nombre (NameError): Surge al utilizar el nombre de una variable que no está definida en el programa.
print(variable_no_definida)
  1. De atributo (AttributeError): Se presenta al intentar obtener o modificar el valor de un atributo inexistente o privado en un objeto o clase.
class MiClase:
    pass

objeto = MiClase()
atributo = objeto.atributo_inexistente
  1. De sintaxis (SyntaxError): Aparece al escribir código que no sigue las reglas del lenguaje de programación utilizado.
z = 10
if z>1
 pass
  1. De entrada/salida de archivo (IOError): Ocurre al intentar leer o escribir en un archivo inexistente o sin los permisos necesarios.
archivo = open("archivo_inexistente.txt", "r")
  1. División por cero (ZeroDivisionError): Se genera al intentar realizar una operación aritmética de división por cero.
resultado = 10 / 0
  1. Etc.

¿Qué son las excepciones?

  • Las excepciones son situaciones de error o eventos inesperados que pueden ocurrir durante la ejecución y que interrumpen el flujo normal de un programa.
  • En Python, una excepción es un mecanismo para declarar y responder a condiciones excepcionales.
    • Python genera una excepción cada vez que ocurre un error.
    • Las excepciones no controladas harán que Python interrumpa la ejecución del programa.
    • Nuestro código puede manejar las excepciones, evitando que la ejecución se detenga.
  • Las excepciones son los objetos. Tienen clases con constructores.

¿Para que sirven las excepciones?

Las excepciones proporcionan información valiosa sobre posibles errores en el código, permitiendo a las/los desarrolladores identificar y gestionar problemas de manera efectiva.

Cada tipo de excepción está diseñado para abordar situaciones específicas, facilitando la depuración y mejorando la robustez de la aplicación.

  • Para garantizar que un programa funcione de manera confiable y no se bloquee inesperadamente.
  • Un manejo efectivo de errores mejora la experiencia del usuario al proporcionar mensajes de error significativos y orientación sobre cómo resolver problemas.
    • Los errores manejados correctamente facilitan la identificación y corrección de problemas durante el desarrollo y el mantenimiento.
    • Un manejo inadecuado de errores puede llevar a vulnerabilidades de seguridad al exponer información sensible o proporcionar a los atacantes información sobre el funcionamiento interno de un sistema.

Manejo de excepciones utilizando bloques try-except:

El manejo de excepciones permite que el programa continúe ejecutándose sin interrupciones, incluso si ocurre un error.

  • El bloque try se utiliza para encerrar el código que podría generar una excepción.
  • El bloque except contiene el código que se ejecutará si se produce la excepción especificada en el bloque try.

a) Instrucción Except sin excepción específica

  • Una instrucción except sin el nombre de excepción específico hará que el intérprete de Python lo considere por defecto como un objeto de la clase base Exception, y capturará todas las excepciones.

  • La sintaxis es la siguiente:

try:
    # Código que puede generar excepciones
except:
    # Código a ejecutar en caso de cualquier excepción
  • En este caso, el bloque except se activará para cualquier tipo de excepción que ocurra dentro del bloque try, proporcionando así una gestión general de errores.
  • Es importante usar esta construcción con precaución, ya que capturar todas las excepciones puede ocultar problemas específicos y dificultar la depuración del código.

b) Instrucción Except con excepción específica

  • Cuando se utiliza el bloque try-except con la especificación de un tipo de excepción, estamos focalizando el manejo de errores en un tipo particular de situación que puede surgir durante la ejecución del código.

  • Esto permite un tratamiento más específico y preciso de las excepciones.

  • La sintaxis es la siguiente:

try:
    # Código que puede generar una excepción específica
except TipoDeExcepcion as nombre:
    # Código a ejecutar en caso de que se produzca la excepción específica

Donde TipoDeExcepcion es el tipo específico de excepción que queremos manejar, y nombre es un identificador opcional que se vincula a la instancia de la excepción para permitir su posterior análisis o manipulación.

Ejemplo

def dividir_numeros():
    try:
        numerador = int(input("Ingrese el numerador: "))
        denominador = int(input("Ingrese el denominador: "))
        resultado = numerador / denominador
        print(f"Resultado de la división: {resultado}")
    except ZeroDivisionError as e:
        print(f"Error: {e}. No se puede dividir por cero.")
        
dividir_numeros()

Manejo de excepciones utilizando bloques try-except:

Flujo de ejecución:

  • El bloque de instrucciones dentro de try se ejecuta primero.
  • Si durante este proceso surge una excepción y si la clase de excepción coincide con la clase especificada en la cláusula except, entonces el bloque de instrucciones dentro de except se ejecuta.
  • En el bloque except, el identificador especificado (e en el ejemplo anterior) se vincula a la excepción generada, lo que permite el acceso a la información sobre la excepción para su posterior procesamiento.

Manejo de múltiples excepciones

  • Python permite manejar varios tipos de excepciones:
    • en un solo bloque except.
    • con varias cláusulas except para diferentes tipos de excepciones.
  • Ejemplos:
def dividir_numeros_v2():
    try:
        numerador = int(input("Ingrese el numerador: "))
        denominador = int(input("Ingrese el denominador: "))
        resultado = numerador / denominador
        print(f"Resultado de la división: {resultado}")
    except (ZeroDivisionError, ValueError) as e:
        print(f"Error: {e}. Ingrese valores numéricos válidos.")

Las instrucciones else y finally

else:

  • El bloque else se ejecuta cuando no se produce ninguna excepción en el bloque try.
  • Proporciona la oportunidad de incluir código que debe ejecutarse solo si no se generan excepciones.

finally:

  • El bloque finally se utiliza para definir código que siempre se ejecutará, independientemente de si se produce una excepción o no.
  • Es útil para realizar acciones de limpieza o liberación de recursos, garantizando que se realicen incluso en casos de excepciones.

Ejemplo

def leer_archivo(nombre_archivo):
    try:
        archivo = open(nombre_archivo, "r")
        contenido = archivo.read()
    except FileNotFoundError:
        print("Error: El archivo no fue encontrado.")
    except IOError:
        print("Error: Hubo un problema al leer el archivo.")
    else:
        # Si no hubo excepciones, mostramos el contenido del archivo
        print("Contenido del archivo:")
        print(contenido)
    finally:
        try:
            archivo.close()  # Cerramos el archivo solo si fue abierto
            print("El archivo ha sido cerrado.")
        except NameError:
            print("No se pudo cerrar el archivo porque no fue abierto correctamente.")

leer_archivo("ejemplo.txt")

Las instrucciones else y finally

  • En este ejemplo, se intenta abrir el archivo ejemplo.txt para su lectura.
  • Si el archivo no se encuentra, se maneja la excepción FileNotFoundError.
  • Si no se produce ninguna excepción, se imprime un mensaje indicando que la lectura fue exitosa.
  • Finalmente, se cierra el archivo en el bloque finally, garantizando que se realice esta acción de limpieza independientemente de si se produjo una excepción o no.

Excepciones personalizadas

Definición y lanzamiento de excepciones:

  • En Python, podemos crear excepciones personalizadas al definir una nueva clase que herede de la clase base Exception o una de sus subclases.
  • Para lanzar una excepción (ya incorporada en Python o personalizada), se usa la palabra reservada raise seguida de una instancia de la clase.
  • Ejemplo:
# Definimos una clase derivada de Exception
class EntradaInvalidaError(Exception):
    def __init__(self, mensaje="Entrada inválida."): 
        super().__init__(mensaje)

# Definimos una función para validar si un correo electrónico tiene el formato correcto
def validar_correo(correo):
    if "@" not in correo:
        raise EntradaInvalidaError("Formato de correo incorrecto.")

# Programa principal
try:
    validar_correo("correo_invalido")
except EntradaInvalidaError as e:
    print(f"Error: {e}")
  • Las excepciones personalizadas son útiles para representar y manejar situaciones específicas en el código que no se ajustan a las excepciones estándar de Python.
